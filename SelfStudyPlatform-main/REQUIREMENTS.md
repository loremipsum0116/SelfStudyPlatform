# REQUIREMENTS

## 1. 프로젝트 개요
이 프로젝트는 Firebase 인증과 Firestore를 사용하는 멀티유저 학습 플랫폼이다.  
현재 과목은 이산수학(`dm`)이며, 챕터별 개념 학습, 일반 연습, 모의고사, 과제풀이, 오답노트 기능을 제공한다.

## 2. 핵심 기능 요구사항

### 2.1 인증
- 모든 학습 페이지는 로그인 후에만 접근 가능해야 한다.
- 미로그인 상태에서 보호 페이지 접근 시 로그인 페이지로 이동해야 한다.

### 2.2 챕터별 연습
- 각 챕터는 `subjects/{subjectId}/questions/chN.json` 문제 파일을 사용한다.
- 일반 연습은 문항별 즉시 채점 방식이어야 한다.
- 사용자가 세션 종료 버튼을 눌렀을 때만 최근 연습 기록을 저장해야 한다.
- 연습문제에서 틀린 문제는 오답노트에 저장하지 않아야 한다.

### 2.3 핵심 문제 필터
- 이산수학 Ch.1~Ch.5 연습 페이지에는 챕터별 핵심 문제 필터가 있어야 한다.
- 필터를 켜면 반복형 문제는 최대한 줄이고, 풀이 로직이 다르거나 헷갈리기 쉬운 문제는 남겨야 한다.
- 각 챕터의 필터 ON/OFF 상태는 별도 preference key로 저장되어야 한다.
- 과제 문제는 필터가 켜져 있어도 항상 표시되어야 한다.
- 과제 문제에는 시각적으로 식별 가능한 라벨이 표시되어야 한다.

### 2.4 챕터별 데이터 반영
- Ch.1은 문제번호/파트 기준 필터를 지원해야 하며, 일부 문제는 본문 값 기반 파트 추론을 지원해야 한다.
- Ch.2는 전 문항이 proof 타입 주관식 구조여야 한다.
- Ch.2의 구형 객관식/진리표 잔여 필드는 제거되어야 한다.
- Ch.3은 whole-problem assignment 예외 처리를 지원해야 한다.
- Ch.4는 데이터 구조상 일부 파트만 분리할 수 없는 경우 문제 전체를 assignment 예외로 처리할 수 있어야 한다.
- Ch.5 13번은 과제용 안내 문구를 포함해야 한다.

### 2.5 모의고사
- 모의고사는 고정 더미 문제 파일에 의존하지 않고, 챕터 문제 파일에서 동적으로 생성해야 한다.
- 사용자는 모의고사 시작 전에 출제할 챕터를 직접 선택할 수 있어야 한다.
- 같은 챕터를 여러 번 선택할 수 있어야 한다.
- 챕터 1회 선택당 6문항을 생성해야 한다.
- 같은 모의고사 세트 안에서는 동일 문항 중복을 최대한 피해야 한다.
- 제출 후 점수, 정답 수, 총 문항 수, 소요 시간, 문항별 판정, 선택 챕터, 생성 요약을 저장해야 한다.
- 최근 기록 화면에는 최근 5회 결과를 표시해야 한다.
- 틀린 문제는 오답노트에 저장해야 한다.

### 2.6 모의고사 출제 규칙
- 각 챕터는 개념 정리 문서의 주제 분류를 기준으로 출제되어야 한다.
- 주제 수가 6개인 챕터는 각 주제에서 1문항씩 출제한다.
- 주제 수가 6개 미만인 챕터는 모든 주제를 최소 1문항씩 포함한 뒤, 주제별 커버 문항 수 비율에 따라 나머지 문항을 배분한다.
- 주제 수가 7개 이상인 챕터는 주제들 중 6개를 무작위로 남기고, 남은 6개 주제에서 1문항씩 출제한다.
- 세부 문항이 많이 쪼개진 챕터라도, 문항 본문에 적힌 원문 문제 번호 기준으로 주제에 매핑해야 한다.

### 2.7 과제풀이
- 별도 페이지 `subjects/{subjectId}/assignment/index.html` 이 존재해야 한다.
- 과제풀이 페이지는 Ch.1~Ch.5에서 과제로 태그된 문제만 자동 수집해야 한다.
- 챕터 구분 없이 한 페이지에서 일괄 풀이할 수 있어야 한다.
- 과제풀이 문제 카드에는 원래 챕터를 식별할 수 있는 태그가 표시되어야 한다.
- 제출 후 점수, 정답 수, 총 문항 수, 소요 시간, 문항별 판정, 챕터 요약을 저장해야 한다.
- 과제풀이 기록은 `mockHistory`와 분리된 `assignmentHistory`에 저장해야 한다.
- 과제풀이에서 틀린 문제는 오답노트에 저장해야 한다.

### 2.8 오답노트
- 오답노트는 과목별로 완전히 분리되어야 한다.
- 저장 대상은 모의고사 오답과 과제풀이 오답이어야 한다.
- 같은 과목에서 같은 문제를 다시 틀리면 중복 문서를 만들지 않고 기존 오답노트 카드에 누적 갱신해야 한다.
- 각 오답 카드에는 `nextReviewAt`이 있어야 하며, 시점 도래 여부에 따라 대기/지금 복습 상태를 구분해야 한다.
- 과목 허브와 오답노트 화면에는 회색(대기) / 빨간색(지금 복습) 배지로 문제 수를 표시해야 한다.
- 복습 성공 누적 횟수에 따라 숙달 상태를 관리해야 한다.
- 오답노트용 question snapshot은 문항 타입에 맞는 필드만 저장해야 하며, 비해당 필드는 빈 배열 또는 `null`로 명시적으로 비워 저장해야 한다.

### 2.9 서술형 입력 보조
- proof/essay 타입 문항에는 서술형 입력 옆에 작은 기호 버튼이 있어야 한다.
- 버튼을 누르면 시그마, 파이, 거듭제곱, 분수, 근호와 주요 논리/집합 기호를 선택할 수 있는 셀렉터가 열려야 한다.
- 시그마·파이 등 구조화된 기호는 세부 값을 입력해 답안 입력창에 자동 삽입할 수 있어야 한다.
- 마이너스 부호가 가능한 구조화 기호는 부호를 선택할 수 있어야 한다.

## 3. 데이터 저장 요구사항

### 3.1 일반 연습
- 경로: `users/{uid}/subjects/{subjectId}/chapters/{chapterId}`
- 저장 필드: `subjectId`, `chapterId`, `correct`, `total`, `score`, `lastDate`, `updatedAt`

### 3.2 모의고사
- 경로: `users/{uid}/subjects/{subjectId}/mockHistory/{autoId}`
- 저장 필드: `score`, `correct`, `total`, `durationMs`, `details`, `chapterSelections`, `generationSummary`, `date`, `createdAt`

### 3.3 과제풀이
- 경로: `users/{uid}/subjects/{subjectId}/assignmentHistory/{autoId}`
- 저장 필드: `score`, `correct`, `total`, `durationMs`, `details`, `chapterSelections`, `generationSummary`, `date`, `createdAt`

### 3.4 오답노트
- 경로: `users/{uid}/subjects/{subjectId}/wrongNotes/{noteId}`
- 저장 필드: `questionId`, `chapterId`, `questionType`, `questionText`, `choices`, `answer`, `accepted`, `headers`, `data`, `answers`, `classificationChoices`, `classificationAnswer`, `explanation`, `hint`, `modelAnswer`, `wrongCount`, `reviewSuccessCount`, `reviewAttemptCount`, `nextReviewAt`, `status`, `lastUserAnswer`, `lastWrongAt`, `lastReviewedAt`, `updatedAt`

## 4. 문서화 요구사항
- README는 실제 현재 구조와 동작을 반영해야 한다.
- 모의고사가 더 이상 `mock.json`을 사용하지 않는다는 점이 문서에 명시되어야 한다.
- 모의고사 주제 매핑과 출제 규칙이 문서에 정리되어 있어야 한다.
- 과제풀이 페이지, assignmentHistory 저장 구조, 오답노트 반영 규칙이 문서에 반영되어야 한다.
- 핵심 문제 필터와 Ch.2 주관식화 반영 내용이 문서에 포함되어야 한다.

## 5. 현재 이산수학 챕터 구성
- `ch1`: 수의 체계와 컴퓨터의 수 표현
- `ch2`: 명제 논리와 추론
- `ch3`: 수학적 증명과 귀납법
- `ch4`: 집합론과 집합의 연산
- `ch5`: 행렬과 행렬 연산
